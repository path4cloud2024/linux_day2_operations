How to increase the disk size (VMDK/VDI) in oracel virtual box?'

(⚠️ TAKE THE SNAPSHOT OF YOUR VM FIRST ⚠️)

1) install below utility before you poweroff the VM
 # dnf install cloud-utils-growpart -y (install growpart)
 # dnf install gdisk -y (install gdisk, sometimes, growpart doesn't work then we can rewrite the partition table wth gdisk utility)
 # snf install testdisk -y (install testdisk, this is being used for recovery of lost filesystems)
 
2) Power off the VM

3) increase the disk size:

	(✅ RECOMMENDE METHOD - VBoxManage modifymedium disk "/path/to/your.vdi" --resize 40960(40GB))

	If doesn't work then follow below:
	a) Open Oracle VM VirtualBox, Go to:
	b) File → Tools → Virtual Media Manager
	c) Select your disk (VDI)
	d) Increase size using slider or input box
	e) Click Apply
	f) restrart the VM	
	
4) login and run lsblk, You’ll see disk increased, but partition NOT yet extended
	(if not visible then run rescan)
	# echo 1 > /sys/class/block/sda/device/rescan
	# lsblk
	NAME                MAJ:MIN RM  SIZE RO TYPE MOUNTPOINTS
	sda                   8:0    0   35G  0 disk				<------------ disk increased
	├─sda1                8:1    0    1M  0 part
	├─sda2                8:2    0  200M  0 part /boot/efi
	├─sda3                8:3    0    1G  0 part /boot			<------------ file system is still shows old size
	└─sda4                8:4    0 	  3G  0 part /				
	sdb                   8:16   0   20G  0 disk
	└─sdb1                8:17   0   20G  0 part
	  └─home_vg-home_lv 253:0    0   20G  0 lvm  /home
	sr0                  11:0    1 1024M  0 rom

5) Extend the partition
	- If using normal partition, then do it with growpart
		BONUS: If growpart not available:
			# fdisk /dev/sda
			# delete and recreate partition with same start sector (⚠️ Safe only if done carefully)

		# growpart /dev/sda 4 (/dev/sda is my disk, 4 is partition on that disk)
		⚠️ IF FAILED, you will see below error:
			# growpart /dev/sda 4 
			GPT PMBR size mismatch (8804351 != 73400319) will be corrected by write. The backup GPT table is not on the end of the device. failed [sfd_dump:1] sfdisk --unit=S --dump /dev/sda sfdisk: cannot open /dev/sda: Input/output error FAILED: failed to dump sfdisk info for /dev/sda 
			(sometimes this error comes, because GPT partition table is still pointing to old size)
		
		✅ Fix : Repair GPT first
			# parted /dev/sda (if any warning or prompt then pass "FIX")
		OR
		Alternatvely, use gdisk:
			# gdisk /dev/sda
			# w (to write the new partition table)
			- then retry the growpart
		
		⚠️ IF growpart still fails: (fallback method and ✅ recommended)
			# fdisk /dev/sda
				Steps:
				p → note start sector of /dev/sda4
				d → delete partition 4
				n → recreate partition 4
				SAME start sector ⚠️
				extend to full disk
				w → write
		
		RUN THIS for sure, once you got sucess from any fix above to extend the file system:		
			# xfs_growfs /
		
	- if using LVM
		# pvresize /dev/sda4
		# lvextent -l +100%FREE -n /dev/mapper/rhel-root
		# xfs_growfs /

⚠️ ⚠️ IF NOTHING WORKS and You’ve hit a bootloader break (GRUB rescue) ⚠️ ⚠️
	While rebooting, on console, you see:
		ERROR:
		no such partition
		grub rescue>

	✅ follow below:
		1) reboot the VM with ISO image in rescue mode
		2) go with option 1
		3) system will mount as /mnt/sysroot
		4) chroot into system # chroot /mnt/sysroot
			⚠️ If /mnt/sysroot is not mounted then try to create the mount point if not available then try to mount
			# mkdir (for /mnt/sysroot, /mnt/sysroot/boot, /mnt/sysroot/boot/efi)
			# mount /dev/sda4 /mnt/sysroot
			# mount /dev/sda3 /mnt/sysroot/boot
			# mount /dev/sda2 /mnt/sysroot/boot/efi
			and then try to mount and go to next steps 
			
			⚠️ if still it doesn't work then: try with test disk (last option, if doesn't work then ⚠️ restore the VM ⚠️)
			# testdisk /dev/sda
				- select disk
				- partition type as "EFI GPT"
				- Analyse - quick search
				- If still possiblility, testdisk will find the partitions
				- if partition found then press "P" to verify the files
				- then write and say "Y"
				- reboot in rescue mode, and follow from chroot steps.
				
		5) verify partition
			# lsblk
			# fdsil -l
			(if all /dev/sda partition visible, then go to next step)
			
		6) If UEFI:
			# grub2-install /dev/sda
			# grub2-mkconfig -o /boot/efi/EFI/redhat/grub.cfg
			Fix the main grub also:
			# grub2-mkconfig -o /boot/efi/EFI/redhat/grub.cfg
			
		7) then try with gdisk /dev/sda (for recovery mode)
			r
			e
			w
			y
			
		8) exit and reboot
